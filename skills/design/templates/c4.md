## L2 — Container

<!-- The inside of the system: apps, services, datastores, queues. For one deployable, each module =
     one Container. The worker / scheduled job gets its own Container — its lifecycle matters. -->

```mermaid
C4Container
    title <feature> — Containers

    Person(actor, "<Actor>")

    Container_Boundary(app, "<Our system>") {
        Container(web, "<Web/UI>", "<technology>", "<purpose>")
        Container(api, "<API/handler>", "<technology>", "<purpose>")
        Container(core, "<Core module>", "<technology>", "<domain logic>")
        Container(worker, "<Background worker>", "<technology>", "<async work>")
    }

    ContainerDb(store, "<Datastore>", "<technology>", "<purpose>")
    System_Ext(extA, "<External system A>", "<purpose>")

    Rel(actor, web, "Uses", "<protocol>")
    Rel(web, api, "Calls", "<protocol>")
    Rel(api, core, "service calls")
    Rel(core, store, "Reads/writes", "<driver>")
    Rel(core, worker, "hands off async work")
    Rel(worker, extA, "Emits", "<protocol>")
```
